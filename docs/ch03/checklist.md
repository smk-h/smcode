# smcode TUI 界面优化 Checklist

> 每一项通过运行代码或观察行为来验证，聚焦系统行为。

## 编译与构建

- [ ] 项目可编译（验证：`npm run build` 退出码为 0，无 TypeScript 错误）
- [ ] 无未引用遗留组件（验证：`grep -R "MessageBubble" src/` 无输出）

## 欢迎页

- [ ] 启动即见顶部欢迎卡（验证：启动 smcode，顶部常驻显示圆角边框欢迎卡，borderText 标题 `smcode vX.X.X`，含 Clawd、模型·Provider、当前目录）
- [ ] 欢迎卡外无第二层边框（验证：欢迎卡之外不再有消息区的白色外框，无双重边框）
- [ ] 欢迎页右侧分栏正确（验证：右侧上方 "Tips for getting started"，下方 "What's new"）
- [ ] 对话时欢迎页常驻（验证：发送消息后欢迎卡仍在顶部，消息在其下方，不被清空）
- [ ] 顶部无独立 smcode 标题栏（验证：原顶部 "smcode / Ctrl+C 退出" 标题框已删除）

## 消息区

- [ ] 用户消息使用深色气泡（验证：用户消息带有深色背景，左侧显示 `❯` 提示符）
- [ ] 用户消息背景铺满整行（验证：深色背景填满终端整行宽度，左右无留白）
- [ ] 用户消息 ❯ 与助手 ● 对齐（验证：`❯`、`●`、完成态 `✻` 三者左边缘在同一列，形成垂直"装订线"；三者各自占用固定 2 字符边沟，内容区从第 2 列起；三者均为中性灰/白色，不染品牌橙）
- [ ] 助手消息带圆点标识（验证：助手消息左侧显示 `●` 圆点）
- [ ] 助手消息支持基础 Markdown（验证：输入包含列表或代码块的问题，助手回复中列表项前有 `-`、代码块被反引号包裹并换行）
- [ ] 完成态行与圆点对齐（验证：`✻ Brewed for Xs` 的 ✻ 与上方 `●` 圆点在同一左边界，不缩进到内容区）
- [ ] 系统消息不显示在主区域（验证：system prompt 不出现在欢迎页或消息列表中）

## 思考过程

- [ ] 思考过程默认折叠（验证：助手思考时显示 `Thinking for Xs (ctrl+o to expand)` 提示，前缀无 `∴`、以空格占位）
- [ ] 思考指示器独立成行且与圆点对齐（验证：思考提示独占一行、渲染在助手 `●` 圆点行之前；前缀（折叠空格 / 展开 `∴`）与 `●` 左边缘在同一列，思考行前没有 `●` 圆点）
- [ ] 按 Ctrl+O 可展开思考（验证：思考过程中或结束后按 Ctrl+O，显示完整 thinking 文本）
- [ ] 再次按 Ctrl+O 可折叠（验证：展开状态下再次按 Ctrl+O，思考文本隐藏）

## 处理中状态

- [ ] 响应期间显示动态提示行（验证：发送消息后、助手内容返回前，消息列表底部出现 `✶ 动词…（耗时 · token · thinking）` 形态的提示行）
- [ ] 图标呼吸脉冲（验证：提示行左侧图标字符随时间循环切换，可见由小到大再回缩的形态/大小变化，而非静止符号）
- [ ] 动词生效与轮换（验证：不同对话轮次可见不同动词；助手执行任务时动词可反映任务进行式；一轮结束后出现完成式，如 `Brewed for Xs`）
- [ ] 思考状态切换（验证：思考时括号内出现 `thinking`，结束后出现 `thought for Ns`，且不会因思考时长过短而一闪而过）
- [ ] 卡顿变红（验证：人为让响应停滞数秒，图标与动词文本逐渐变红；恢复输出后颜色回到主色）
- [ ] token 平滑增长（验证：流式输出时 token 数平滑递增，而非跳变）
- [ ] 状态行宽度自适应（验证：缩小终端窗口，提示行优先保留图标与动词，token/耗时按宽度逐级降级消失，不换行、不撑爆）
- [ ] 响应结束后状态行消失（验证：流式输出完成后，状态行不再显示）

## 底部输入框

- [ ] 输入框为上下两条灰白边界线（验证：输入框上下各一条灰白直线，无左右边框）
- [ ] 左侧显示 `❯` 提示符（验证：输入框左侧有 `❯` 字符）
- [ ] 底部显示快捷提示（验证：输入框下方可见 `Enter 发送` 或 `esc to interrupt` 等提示）

## 现有功能无回归

- [ ] Provider 选择正常（验证：配置多个 Provider 时启动进入选择界面，方向键与 Enter 可用）
- [ ] 流式输出正常（验证：发送消息后助手回复逐字/逐块出现）
- [ ] Token 统计正常（验证：多轮对话后 token 数累加，状态行或界面可见累计值）
- [ ] Ctrl+C 退出正常（验证：任意界面按 Ctrl+C 可退出程序）
- [ ] Enter 发送正常（验证：输入文字后按 Enter 发送消息）

## 端到端场景

- [ ] 场景 1：首次启动 → 看到欢迎页 → 发送 "你好" → 看到用户深色气泡 → 看到思考提示（`Thinking…`、无 `∴`）→ 看到助手回复 → 按 Ctrl+O 展开思考 → 看到 `∴` 前缀的完整思考文本
- [ ] 场景 2：连续对话 → 第二轮发送消息 → 历史消息保留 → 新的助手回复正常出现 → 输入框在等待回复时变暗并显示中断提示
- [ ] 场景 3：发送一个需要较长思考/输出的问题 → 看到图标字符呼吸脉冲（形态/大小变化）→ 括号内先显示 `thinking` 再显示 `thought for Ns` → token 平滑增长 → 中途人为停滞数秒可见图标与动词变红 → 结束后出现 `Brewed for Xs` 完成式
