# smcode 第 1 章：交互式对话 Checklist

> 每一项通过运行代码或观察行为来验证，聚焦系统行为。

## 实现完整性

- [ ] **TUI 正常启动**（对应 AC1）：执行 `npm run build && npm start`，观察到终端进入对话界面，包含标题栏、对话展示区和底部输入区，应用未异常退出。
- [ ] **用户输入可见**（对应 AC2）：在输入区键入 `你好` 并按 Enter，观察到 `你好` 以用户消息形式出现在对话展示区。
- [ ] **多轮历史生效**（对应 AC3）：
  1. 发送 `请记住我的名字是 Alice`。
  2. 发送 `我刚才让你记住了什么名字？`。
  3. 观察到助手回答中包含 `Alice`，证明历史上下文被传入模型。
- [ ] **流式渲染**（对应 AC4）：发送一个需要多 token 回答的问题（如 `请用 200 字介绍自己`），观察到答案文字逐段出现，而非长时间停顿后一次性完整出现。
- [ ] **OpenAI 后端可用**（对应 AC5）：使用 `protocol: openai` 的配置，发送消息，观察到正常流式回复。
- [ ] **思考过程展示**（对应 AC6）：使用一个会返回 `reasoning_content` 的 OpenAI 兼容后端（如 DeepSeek 推理模型），发送需要推理的问题，观察到 TUI 先展示思考过程，再展示最终答案。
- [ ] **Token 消耗展示**（对应 AC7）：在一次完整对话后，观察到 TUI 底部状态栏以纯文字形式显示本次对话累计的 `prompt: X  completion: Y  total: Z`。
- [ ] **Provider 抽象保留扩展能力**（对应 AC5）：在不修改 TUI 和会话模块源码的情况下，能够通过新增 Provider 实现类并在工厂中注册来扩展新后端（可通过代码审查确认接口与工厂结构）。
- [ ] **YAML 配置读取**（对应 AC8）：在 `.smcode/config.yaml` 中填写四个字段后启动应用，观察到配置被正确用于调用模型（无配置格式错误）。
- [ ] **自定义配置路径**（对应 AC9）：执行 `node bin/smcode.js --config ./custom.yaml`，观察到应用使用 `custom.yaml` 而不是 `.smcode/config.yaml`。
- [ ] **错误可读性**（对应 AC10）：
  - 临时删除 `.smcode/config.yaml`，启动应用，观察到清晰的错误提示（如配置文件不存在）。
  - 恢复配置但填写错误 API key，发送消息，观察到界面展示可理解的认证/请求错误，应用未崩溃。

## 集成

- [ ] **Provider 抽象生效**：在不修改 `src/tui/app.tsx` 和 `src/conversation/session.ts` 的情况下，仅通过替换 Provider 实例即可改变实际调用的后端（后续扩展时验证）。
- [ ] **会话模块正确驱动 Provider**：多轮对话中，每一轮请求的消息列表都包含此前所有用户与助手消息，且 thinking 内容不会被作为上下文回传（可通过在 Provider 实现中临时打印消息列表验证）。
- [ ] **流式数据正确关闭**：对话结束后，助手消息不再显示「思考中...」或等待占位符，且最终内容与思考内容均完整。

## 编译与测试

- [ ] **项目编译无错误**：执行 `npm run build`，`out/` 目录生成所有对应 `.js` 与 `.d.ts`，终端无 TypeScript 类型错误。
- [ ] **类型检查通过**：执行 `npx tsc --noEmit`，返回退出码 0。
- [ ] **smink 单实例**：执行 `npm ls react`，确认只存在一个 react 版本，避免双实例导致的 Hook 错误。
- [ ] **代码符合 ts-lang-spec**：关键模块（`src/provider/*`、`src/conversation/*`、`src/tui/*`、`src/index.ts`）的命名、注释、类型导入符合 TypeScript 规范技能要求。

## 端到端场景

- [ ] **场景 1：OpenAI 兼容后端完整对话**：
  - 准备 `.smcode/config.yaml`，配置一个可用的 OpenAI 兼容后端。
  - 启动 `npm start`。
  - 输入 `请用一句话打招呼`，按 Enter。
  - 预期：界面流式显示助手回复，底部状态栏以纯文字显示本次累计 token 数量（如 `prompt: X  completion: Y  total: Z`），且回复内容符合提问。

- [ ] **场景 2：思考过程展示**：
  - 准备使用会返回 `reasoning_content` 的 OpenAI 兼容后端配置文件（如 DeepSeek 推理模型）。
  - 启动应用并指定该配置。
  - 输入 `9.11 和 9.9 哪个更大？请逐步思考`。
  - 预期：界面先显示思考过程，再显示最终结论，结论正确，底部状态栏以纯文字显示 token 数量并随回复增加。

- [ ] **场景 3：连续上下文对话**：
  - 启动应用。
  - 第一轮：`我叫 Bob`。
  - 第二轮：`1 + 1 等于几？`。
  - 第三轮：`请说出我的名字和刚才的算式答案`。
  - 预期：助手同时提到 `Bob` 和 `2`，证明多轮历史完整保留。

- [ ] **场景 4：异常恢复**：
  - 启动应用后，先发送一条正常消息并收到回复。
  - 临时断开网络或填入错误 API key，再发送一条消息。
  - 预期：第二条消息以错误提示展示，应用仍可继续输入和发送下一条消息。
